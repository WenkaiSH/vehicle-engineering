# 自动驾驶域控制器(ADC)架构

## 1. ADC系统概述

### 1.1 ADC定义与定位

自动驾驶域控制器(Autonomous Driving Controller, ADC)是智能驾驶系统的核心计算单元，负责感知、融合、决策、规划全流程处理。

```
ADC在整车架构中的位置:

┌─────────────────────────────────────────────────────────────┐
│                        中央计算大脑                          │
│              ┌─────────────────────────────┐                │
│              │     自动驾驶域 (ADC)        │                │
│              │  ┌───────────────────────┐  │                │
│              │  │   感知 → 融合 → 规划   │  │                │
│              │  │   ↓                    │  │                │
│              │  │   决策 → 控制 → 车辆   │  │                │
│              │  └───────────────────────┘  │                │
│              └─────────────────────────────┘                │
│                            │                                │
│              ┌─────────────┼─────────────┐                  │
│              ▼             ▼             ▼                  │
│         ┌────────┐   ┌────────┐   ┌────────┐               │
│         │ 传感器 │   │ 执行器 │   │ 座舱域 │               │
│         └────────┘   └────────┘   └────────┘               │
└─────────────────────────────────────────────────────────────┘

ADC核心职责:
1. 多传感器数据接收与同步
2. 环境感知 (目标检测、跟踪、预测)
3. 高精定位与建图
4. 路径规划与行为决策
5. 车辆控制指令生成
6. 功能安全与冗余管理
```

### 1.2 ADC发展阶段

| 阶段 | 算力 | 传感器 | 功能 | 代表平台 |
|-----|------|--------|------|---------|
| **L2辅助** | 5-30 TOPS | 5V1R | ACC/LKA | EyeQ4/Xavier |
| **L2+高速NOA** | 30-100 TOPS | 7-11V1R1L | 高速领航 | Orin-X/J5 |
| **L2++城市NOA** | 100-500 TOPS | 11V5R1-2L | 城市领航 | 2×Orin-X |
| **L3条件自动驾驶** | 200-1000 TOPS | 多V多R多L | 特定场景脱手 | 冗余架构 |
| **L4高度自动驾驶** | 1000+ TOPS | 全覆盖 | 无人驾驶 | Robotaxi |

## 3. 感知系统架构

### 3.1 多传感器融合架构

```
传感器融合架构演进:

传统架构 (后融合):
摄像头 ──▶ 目标检测 ──┐
                     ├──▶ 融合层 ──▶ 跟踪 ──▶ 规划
雷达 ────▶ 目标检测 ──┘

BEV架构 (特征级融合):
摄像头×N ──┐
           ├──▶ BEV编码器 ──▶ BEV特征图 ──▶ 检测头 ──▶ 规划
雷达×M ────┘

端到端架构:
传感器数据 ──▶ 统一骨干网络 ──▶ 端到端输出(轨迹/控制)

当前主流: BEV+Transformer架构
```

### 3.2 BEV (Bird's Eye View) 感知

| 模块 | 功能 | 技术方案 |
|-----|------|---------|
| **图像编码** | 提取2D特征 | ResNet/ViT |
| **视角转换** | 2D→3D/BEV | LSS/Transformer/MLP |
| **BEV编码** | 时序融合 | 多帧BEV对齐 |
| **检测头** | 目标检测 | Anchor-free/Anchor-based |
| **跟踪** | 目标跟踪 | MOT/MOTR |

**BEV视角转换方法对比**:

| 方法 | 原理 | 优点 | 缺点 |
|-----|------|------|------|
| **LSS (Lift-Splat-Shoot)** | 深度估计+视锥体转换 | 显式深度，可解释 | 深度误差敏感 |
| **Transformer** | Cross-attention隐式学习 | 无需显式深度 | 计算量大 |
| **MLP/FC** | 全连接映射 | 简单快速 | 泛化性一般 |
| **几何投影** | 相机标定投影 | 精确 | 依赖标定精度 |

## 4. 决策规划系统

### 4.1 规划架构分层

```
分层规划架构:

┌─────────────────────────────────────────────────────────────┐
│                    行为决策层 (Behavior)                     │
│               决定做什么 (What to do)                        │
│         ┌─────────────────────────────────┐                 │
│         │ 场景理解 → 意图预测 → 行为决策   │                 │
│         │ 换道/超车/跟车/停车/绕行...      │                 │
│         └─────────────────────────────────┘                 │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                    路径规划层 (Path Planning)                │
│               决定走哪条路 (Which way)                        │
│         ┌─────────────────────────────────┐                 │
│         │ 全局路径 → 局部路径 → 轨迹优化   │                 │
│         │ A*/Hybrid A* + QP优化           │                 │
│         └─────────────────────────────────┘                 │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                    运动规划层 (Motion Planning)              │
│               决定怎么走 (How to go)                          │
│         ┌─────────────────────────────────┐                 │
│         │ 速度规划 → 轨迹平滑 → 控制指令   │                 │
│         │ 考虑舒适性、安全性、效率          │                 │
│         └─────────────────────────────────┘                 │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 决策方法对比

| 方法 | 原理 | 优点 | 缺点 | 应用 |
|-----|------|------|------|------|
| **规则-based** | 专家规则+状态机 | 可解释性强 | 复杂场景难覆盖 | L2辅助 |
| **强化学习** | 奖励驱动学习 | 适应复杂场景 | 训练困难，安全难保证 | 研究阶段 |
| **模仿学习** | 模仿人类驾驶 | 数据驱动 | 分布偏移问题 | 研究阶段 |
| **混合方法** | 规则+学习结合 | 兼顾安全与智能 | 复杂度高 | 主流方案 |

### 4.3 规划算法

**路径规划算法对比**:

| 算法 | 适用场景 | 计算复杂度 | 最优性 | 平滑性 |
|-----|---------|-----------|--------|--------|
| **A*** | 全局路径 | O(b^d) | 最优 | 差 |
| **Hybrid A*** | 车辆运动学 | 中等 | 次优 | 中等 |
| **RRT*** | 高维空间 | 中等 | 渐近最优 | 中等 |
| **Lattice** | 结构化道路 | 低 | 次优 | 好 |
| **Optimization (QP)** | 局部优化 | 低 | 局部最优 | 好 |

**当前主流方案**: Hybrid A* + Lattice采样 + QP优化

```
功能安全架构 (ASIL-D):

安全监控单元 (Safety MCU):
┌─────────────────────────────────────────────────┐
│  ┌──────────┐  ┌──────────┐  ┌──────────┐      │
│  │ SoC监控  │  │ 传感器校验│  │ 仲裁表决 │      │
│  │ 看门狗   │  │ 冗余对比 │  │ 主/副切换│      │
│  └──────────┘  └──────────┘  └──────────┘      │
├─────────────────────────────────────────────────┤
│  故障响应策略:                                  │
│  Level 1: 报警提示 → 请求驾驶员接管            │

- 双核锁步 (Lock-step)

```

### 6.2 SOTIF (预期功能安全)

| 风险类型 | 描述 | 缓解措施 |
|---------|------|---------|
| **性能不足** | 传感器在恶劣天气失效 | 多传感器冗余 |
| **误触发** | 误识别导致紧急制动 | 多帧确认、置信度评估 |
| **人机误解** | 驾驶员不理解系统限制 | HMI明确提示 |
| **复杂场景** | 非标准道路场景 | 场景库覆盖、仿真测试 |
