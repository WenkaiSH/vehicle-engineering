# 执行器与线控系统

## 1. 线控系统概述

### 1.1 线控技术定义

线控技术 (Drive-by-Wire, X-by-Wire) 是指通过电信号替代传统的机械或液压连接，实现车辆控制指令的传输与执行。

```
线控系统演进对比:

传统机械/液压连接              线控电信号连接
┌──────────────────┐          ┌──────────────────┐
│  驾驶员操作      │          │  驾驶员操作      │
│   (方向盘)       │          │   (方向盘/屏幕)  │
└────────┬─────────┘          └────────┬─────────┘
         │                             │
         ▼                             ▼
┌──────────────────┐          ┌──────────────────┐
│  机械传动轴      │          │  传感器检测      │
│  转向柱/传动杆   │          │  扭矩/角度/力    │
└────────┬─────────┘          └────────┬─────────┘
         │                             │
         ▼                             ▼
┌──────────────────┐          ┌──────────────────┐
│  液压助力系统    │          │  ECU处理         │
│  助力泵/油管    │          │  信号转换        │
└────────┬─────────┘          └────────┬─────────┘
         │                             │
         ▼                             ▼
┌──────────────────┐          ┌──────────────────┐
│  执行机构        │          │  电信号传输      │
│  转向机/制动钳   │          │  CAN/Ethernet   │
└──────────────────┘          └────────┬─────────┘
                                       │
                                       ▼
                              ┌──────────────────┐
                              │  电机执行机构    │
                              │  线控电机+减速器 │
                              └──────────────────┘

线控三大核心系统:
├─ 线控转向 (Steer-by-Wire)
├─ 线控制动 (Brake-by-Wire)
└─ 线控驱动 (Drive-by-Wire / Throttle-by-Wire)
```

### 1.2 线控系统优势

| 优势 | 具体表现 | 自动驾驶意义 |
|-----|---------|-------------|
| **响应速度** | 电信号毫秒级响应 | 满足自动驾驶实时性 |
| **精度控制** | 电机控制精度高 | 实现精确轨迹跟踪 |
| **设计灵活** | 取消机械连接 | 布置自由，空间优化 |
| **功能扩展** | 易于集成新功能 | 支持OTA升级 |
| **安全冗余** | 双电机/双电控 | 满足ASIL-D要求 |
| **能耗优化** | 按需供能 | 提升续航里程 |

### 1.3 线控系统安全等级

```
线控系统功能安全要求:

系统            安全等级       冗余要求
─────────────────────────────────────────
线控转向         ASIL-D        双电机+双ECU
线控制动         ASIL-D        双回路+双ECU
线控驱动         ASIL-C/D      双ECU监控

失效安全策略 (Fail-Operational):
├─ 单点故障 → 系统继续正常运行
├─ 双点故障 → 系统进入安全模式
└─ 故障检测 → 100ms内响应
```

### 2.2 线控转向 vs EPS

| 特性 | EPS电动助力 | 线控转向SbW | 差异 |
|-----|------------|------------|------|
| **机械连接** | 保留转向柱 | 完全取消 | 核心区别 |
| **路感反馈** | 机械传递 | 电机模拟 | 可编程 |
| **转向比** | 固定 | 可变 | 灵活调节 |
| **布置空间** | 受限 | 自由 | 节省空间 |
| **失效安全** | 机械备份 | 双冗余 | 更复杂 |
| **法规** | 成熟 | 逐步开放 | 新兴技术 |
| **成本** | 低 | 高 | 2-3倍 |

### 5.2 安全设计原则

| 层级 | 措施 | 目标 |
|-----|------|------|
| **硬件冗余** | 双ECU、双电机、双传感器 | 单点故障不影响功能 |
| **通信冗余** | 双总线、双电源 | 通信链路冗余 |
| **软件监控** | 看门狗、程序流监控 | 软件故障检测 |
| **故障响应** | 降级策略、安全停车 | 故障后安全状态 |
