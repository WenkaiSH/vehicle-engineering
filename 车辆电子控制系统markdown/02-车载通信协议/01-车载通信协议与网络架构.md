| **FlexRay** | 10Mbps | 极高 | 高 | 线控制动 | 被TSN替代 |
| **Ethernet** | 100Mbps-10Gbps | 可配置 | 中 | ADAS/座舱 | 成为主干 |
| **TSN** | 100Mbps-10Gbps | 确定性 | 中 | 下一代主干 | 2025+主流 |

```
      │ (GW)    │
      └────┬────┘
           │
    ┌──────┼──────┐
    │      │      │
    ▼      ▼      ▼
┌──────┐┌──────┐┌──────┐
│ECU-1 ││ECU-2 ││ECU-3 │
│发动机 ││变速箱 ││ABS   │
└──────┘└──────┘└──────┘

CAN帧结构:
┌────────┬────────┬────────┬────────┐
│  SOF   │ 仲裁场 │ 控制场 │ 数据场 │
│(1bit)  │(12bit) │(6bit)  │(0-8B)  │
├────────┼────────┼────────┼────────┤

```

### 2.2 CAN vs CAN-FD 对比

| 特性 | CAN 2.0 | CAN-FD | 提升 |
|-----|---------|--------|------|
| **数据速率** | 500kbps-1Mbps | 2-5Mbps | 4-10倍 |
| **数据长度** | 8字节 | 64字节 | 8倍 |
| **帧格式** | 标准/扩展 | 标准/扩展 + FD | 兼容 |
| **CRC** | 15位 | 17/21位 | 更强 |
| **延迟** | ~100μs | ~25μs | 75%↓ |

### 2.3 CAN-FD报文示例

```

      └─ 转速: 400 RPM (0x0190)
      └─ 扭矩: 10 Nm (0x0A)
      └─ 温度: 后续字节

周期: 10ms
优先级: 高 (ID小)
```

## 4. TSN (时间敏感网络)

### 4.1 TSN核心技术

TSN是IEEE 802.1工作组定义的一系列标准，为以太网提供确定性传输。

| 标准 | 功能 | 说明 |
|-----|------|------|
| **802.1AS** | 时间同步 | gPTP精密时间协议，精度<1μs |
| **802.1Qbv** | 流量整形 | 门控调度，时间触发传输 |
| **802.1Qbu** | 帧抢占 | 高优先级打断低优先级帧 |
| **802.1Qav** | 信用整形 | CBS保证带宽分配 |
| **802.1CB** | 帧复制消除 | 冗余传输，提高可靠性 |
| **802.1Qci** | 入口过滤 | 防止网络拥塞和攻击 |

### 4.2 TSN时间触发调度

```
TSN门控调度示意 (802.1Qbv):

时间轴(ms):    0    0.5   1.0   1.5   2.0
               │     │     │     │     │
               ▼     ▼     ▼     ▼     ▼
队列7(最高):   ┌─┐   ┌─┐   ┌─┐   ┌─┐   ┌─┐
              │█│   │ │   │█│   │ │   │█│  控制信号
队列6:         ┌─┐   ┌─┐   ┌─┐   ┌─┐   ┌─┐
              │ │   │█│   │ │   │█│   │ │  传感器数据
队列5:         ┌─┐   ┌─┐   ┌─┐   ┌─┐   ┌─┐
              │█│   │ │   │█│   │ │   │█│  视频流
队列0(最低):   ┌─┐   ┌─┐   ┌─┐   ┌─┐   ┌─┐
              │ │   │ │   │ │   │ │   │ │  诊断/日志

门控列表(GCL):
T0: 队列7打开 → 发送紧急控制帧
T0.5: 队列6打开 → 发送传感器数据
T1.0: 队列7打开 → 发送紧急控制帧
...

关键参数:
- 周期: 1ms
- 精度: ±100ns
- 抖动: <1μs
```

### 4.3 TSN vs 传统以太网

| 特性 | 标准以太网 | TSN | 提升 |
|-----|-----------|-----|------|
| **实时性** | 尽力而为 | 确定性 | 有界延迟 |
| **时间同步** | 无 | ±1μs | 精密同步 |
| **延迟** | 不确定 | 可预测 | 99.999%确定性 |
| **可靠性** | 单路径 | 冗余路径 | 零丢包 |
| **适用性** | IT网络 | 工业/汽车 | 安全关键 |

## 5. DDS (数据分发服务)

### 5.1 DDS核心概念

DDS是OMG制定的发布-订阅中间件标准，面向实时分布式系统。

```
DDS发布-订阅模型:

┌──────────────────────────────────────────────────────────┐
│                    DDS域 (Domain)                        │
├──────────────────────────────────────────────────────────┤
│                                                          │
│   ┌──────────┐        ┌──────────┐        ┌──────────┐  │
│   │  Publisher│        │  DDS总线  │        │ Subscriber│  │
│   │  (发布者) │───────▶│  (全局    │───────▶│  (订阅者) │  │
│   │           │        │  数据空间) │        │           │  │
│   │ 摄像头节点│        │           │        │ 融合节点  │  │
│   └──────────┘        │           │        └──────────┘  │
│                        │           │                      │
│   ┌──────────┐        │           │        ┌──────────┐  │
│   │  Publisher│───────▶│           │───────▶│ Subscriber│  │
│   │ 激光雷达  │        │           │        │ 规划节点  │  │
│   └──────────┘        │           │        └──────────┘  │
│                        │           │                      │
│   ┌──────────┐        │           │        ┌──────────┐  │
│   │  Publisher│───────▶│           │───────▶│ Subscriber│  │
│   │ 毫米波雷达│        │           │        │ HMI节点   │  │
│   └──────────┘        └──────────┘        └──────────┘  │
│                                                          │
└──────────────────────────────────────────────────────────┘

特点:
- 去中心化: 无单点故障
- 动态发现: 节点自动发现
- QoS策略: 灵活配置传输质量
```

### 5.2 DDS QoS策略

| QoS策略 | 功能 | 应用场景 |
|--------|------|---------|
| **Reliability** | 可靠/最佳 effort | 控制指令必须可靠 |
| **Durability** | 历史数据保留 | 新订阅者获取历史 |
| **Deadline** | 数据时效性 | 传感器数据超时告警 |
| **Latency Budget** | 延迟预算 | 保证端到端延迟 |
| **Ownership** | 多发布者仲裁 | 主备切换 |
| **Lifespan** | 数据有效期 | 过期数据自动清理 |
| **History** | 缓存策略 | Keep-last/Keep-all |

### 5.3 DDS vs SOME/IP 对比

| 维度 | DDS | SOME/IP | 说明 |
|-----|-----|---------|------|
| **架构** | 发布-订阅 | 服务发现+RPC | DDS更灵活 |
| **发现机制** | 动态发现 | 静态配置+服务发现 | DDS自适应 |
| **QoS** | 丰富(22+种) | 有限 | DDS更精细 |
| **实时性** | 微秒级 | 毫秒级 | DDS更实时 |
| **资源占用** | 较高 | 较低 | SOME/IP轻量 |
| **AUTOSAR** | Adaptive支持 | Classic+Adaptive | 后者更成熟 |
| **生态** | 工业/军工 | 汽车行业 | 后者车规化 |

## 7. 网络安全

### 7.1 车载网络安全威胁

| 攻击向量 | 威胁 | 防护措施 |
|---------|------|---------|
| **OBD接口** | 物理接入攻击 | 认证、防火墙 |
| **T-Box** | 远程攻击入口 | IDS/IPS、安全网关 |
| **OTA通道** | 固件篡改 | 签名验证、加密 |
| **诊断接口** | 未授权诊断 | 诊断防火墙 |
| **V2X通信** | 消息伪造 | PKI认证、消息签名 |

### 7.2 安全机制

```
车载网络安全分层:

应用层:
├─ 应用认证 (TLS/mTLS)
├─ 数据加密 (AES-256)
└─ 消息签名 (HMAC/ECDSA)

传输层:
├─ 安全通信 (DTLS/TLS 1.3)
├─ 防火墙规则
└─ 入侵检测 (IDS)

网络层:
├─ 网络隔离 (VLAN/防火墙)
├─ 安全网关 (中央网关)
└─ 流量监控 (异常检测)

物理层:
├─ 接口保护 (OBD锁止)
├─ 调试端口禁用
└─ 安全启动 (Secure Boot)
```
