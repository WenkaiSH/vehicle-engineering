# 车载计算平台与AI芯片

## 1. 算力需求分析

### 1.1 自动驾驶算力增长曲线

```
算力需求增长趋势 (TOPS):

L2+        L2++       L3         L4         L5
  │          │          │          │          │
 10         30        100        500       2000
  │          │          │          │          │
  ▼          ▼          ▼          ▼          ▼
Mobileye   Xavier    Orin-X     Thor       自研
EyeQ4     (30T)    (254T×4)   (2000T)    芯片
(2.5T)                          2025       待定
2018      2020       2022                  

关键驱动因素:
├─ 摄像头分辨率: 1MP → 8MP → 12MP
├─ 激光雷达: 无 → 1-2颗 → 4-6颗
├─ 融合算法复杂度: 2D → 3D → BEV+Transformer → VLA
└─ 功能场景: 高速NOA → 城市NOA → 全场景 → L4
```

### 1.2 不同级别自动驾驶算力需求

| 功能场景 | 传感器配置 | 算力需求 | 典型芯片 | 代表车型 |
|---------|-----------|---------|---------|---------|
| **高速NOA** | 5V1R | 10-30 TOPS | EyeQ4/Xavier/J6E | 理想ONE/比亚迪秦 |
| **城市NOA** | 11V1R1L | 100-200 TOPS | Orin-X/J6M/J6P | 小鹏G9/蔚来ET5 |
| **L3高速** | 11V5R1L | 200-500 TOPS | 2×Orin-X/J6P | 蔚来ET7/理想L9 MAX |
| **L4 Robotaxi** | 多V多R多L | 1000+ TOPS | Thor/双J6P | Waymo/百度Apollo |
| **舱驾一体** | 融合方案 | 500-2000 TOPS | Thor/J6P+8295 | 极氪007/零跑D平台 |

### 1.3 算力消耗分布

| 任务类型 | 算力占比 | 说明 |
|---------|---------|------|
| **感知** | 60-70% | 目标检测、语义分割、BEV构建 |
| **融合** | 10-15% | 多传感器时空融合 |
| **规划** | 10-15% | 路径规划、行为预测 |
| **控制** | 5-10% | 轨迹跟踪、车辆控制 |

### 1.4 端到端大模型对算力的新要求

```
传统架构 vs 端到端大模型算力对比:

传统分模块架构:
感知(CNN) ──┐
            ├──▶ 融合 ──▶ 规划 ──▶ 控制
定位(SLAM) ─┘
总算力: 100-200 TOPS

端到端大模型架构:
原始传感器数据 ──▶ Transformer大模型 ──▶ 直接输出控制信号
                    (VLA: Vision-Language-Action)
总算力: 500-2000 TOPS (增加5-10倍)

关键变化:
├─ Transformer计算需求: O(n²)复杂度，n=token数
├─ 大模型参数量: 1B → 10B → 100B
├─ 实时性要求: <100ms延迟
└─ 内存带宽: 从200GB/s提升到1TB/s+
```

## 3. 异构计算架构

### 3.1 异构计算单元分工

| 计算单元 | 职责 | 适用任务 | 典型性能 |
|---------|------|---------|---------|
| **CPU** | 逻辑控制、决策 | 路径规划、行为预测 | 100-200K DMIPS |
| **GPU** | 并行计算、AI推理 | 感知算法、BEV | 100-500 TOPS |
| **NPU/AI加速器** | 深度学习推理 | CNN/Transformer | 100-1000 TOPS |
| **ISP** | 图像预处理 | 去噪、HDR、畸变校正 | 1-2GPix/s |
| **DSP** | 信号处理 | 雷达信号、语音 | 高效FFT |
| **VPU** | 视频编解码 | 舱内监控、记录 | 4K@60fps |

### 3.2 异构计算调度

```
异构计算任务调度:

摄像头数据 ──▶ ISP ──▶ 图像预处理 ──┬──▶ NPU (目标检测)
                                   ├──▶ NPU (语义分割)
激光雷达 ────▶ DSP ──▶ 点云处理 ────┼──▶ NPU (点云检测)
                                   │
                                   ▼
                           融合层 (GPU/CPU)
                                   │
                                   ▼
                           规划决策 (CPU)
                                   │
                                   ▼
                           控制输出 (安全MCU)

数据流优化:
- 零拷贝: 减少内存拷贝开销
- 流水线: 并行处理多帧数据
- 动态调度: 根据负载调整核心分配
```

## 5. 热设计与可靠性

### 5.1 芯片散热方案
