# 车载计算平台与AI芯片

## 1. 算力需求分析

### 1.1 自动驾驶算力增长曲线

```
算力需求增长趋势 (TOPS):

L2+        L2++       L3         L4         L5
  │          │          │          │          │
 10         30        100        500       2000
  │          │          │          │          │
  ▼          ▼          ▼          ▼          ▼
Mobileye   Xavier    Orin-X     Thor       自研
EyeQ4     (30T)    (254T×4)   (2000T)    芯片
(2.5T)                          2025       待定
2018      2020       2022                  

关键驱动因素:
├─ 摄像头分辨率: 1MP → 8MP → 12MP
├─ 激光雷达: 无 → 1-2颗 → 4-6颗
├─ 融合算法复杂度: 2D → 3D → BEV+Transformer
└─ 功能场景: 高速NOA → 城市NOA → 全场景
```

### 1.2 不同级别自动驾驶算力需求

| 功能场景 | 传感器配置 | 算力需求 | 典型芯片 | 代表车型 |
|---------|-----------|---------|---------|---------|
| **高速NOA** | 5V1R | 10-30 TOPS | EyeQ4/Xavier | 理想ONE |
| **城市NOA** | 11V1R1L | 100-200 TOPS | Orin-X | 小鹏G9 |
| **L3高速** | 11V5R1L | 200-500 TOPS | 2×Orin-X | 蔚来ET7 |
| **L4 Robotaxi** | 多V多R多L | 1000+ TOPS | Thor/自研 | Waymo |

### 1.3 算力消耗分布

| 任务类型 | 算力占比 | 说明 |
|---------|---------|------|
| **感知** | 60-70% | 目标检测、语义分割、BEV构建 |
| **融合** | 10-15% | 多传感器时空融合 |
| **规划** | 10-15% | 路径规划、行为预测 |
| **控制** | 5-10% | 轨迹跟踪、车辆控制 |

## 3. 异构计算架构

### 3.1 异构计算单元分工

| 计算单元 | 职责 | 适用任务 | 典型性能 |
|---------|------|---------|---------|
| **CPU** | 逻辑控制、决策 | 路径规划、行为预测 | 100-200K DMIPS |
| **GPU** | 并行计算、AI推理 | 感知算法、BEV | 100-500 TOPS |
| **NPU/AI加速器** | 深度学习推理 | CNN/Transformer | 100-1000 TOPS |
| **ISP** | 图像预处理 | 去噪、HDR、畸变校正 | 1-2GPix/s |
| **DSP** | 信号处理 | 雷达信号、语音 | 高效FFT |

### 3.2 异构计算调度

```
异构计算任务调度:

摄像头数据 ──▶ ISP ──▶ 图像预处理 ──┬──▶ NPU (目标检测)
                                   ├──▶ NPU (语义分割)
激光雷达 ────▶ DSP ──▶ 点云处理 ────┼──▶ NPU (点云检测)
                                   │
                                   ▼
                           融合层 (GPU/CPU)
                                   │
                                   ▼
                           规划决策 (CPU)
                                   │
                                   ▼
                           控制输出 (安全MCU)

数据流优化:
- 零拷贝: 减少内存拷贝开销
- 流水线: 并行处理多帧数据
- 动态调度: 根据负载调整核心分配
```

## 5. 热设计与可靠性

### 5.1 芯片散热方案

| 方案 | 散热能力 | 成本 | 应用 |
|-----|---------|------|------|
| **自然冷却** | <15W | 低 | 低算力ECU |
| **风冷** | 15-50W | 中 | 座舱域控 |
| **水冷** | 50-150W | 高 | 高算力ADC |
| **相变冷却** | >150W | 很高 | 下一代 |

### 5.2 功能安全等级

| 系统 | ASIL等级 | 要求 | 芯片特性 |
|-----|---------|------|---------|
| 信息娱乐 | QM | 质量管理 | 无特殊要求 |
| 仪表显示 | ASIL-B | 部分功能安全 | ECC内存 |
| 座舱域控 | ASIL-B | 部分功能安全 | Lock-step CPU |
| 自动驾驶 | ASIL-D | 最高安全等级 | 双芯锁步、冗余 |

## 6. 发展趋势

### 6.1 2025-2030技术路线

| 年份 | 趋势 | 技术特征 |
|-----|------|---------|
| **2025** | 舱驾融合 | 一颗芯片同时处理智驾+座舱 |
| **2026** | Chiplet封装 | 模块化设计，灵活配置算力 |
| **2027** | 存算一体 | 减少数据搬运，能效提升10倍 |
| **2028** | 光子计算 | 光互连突破带宽瓶颈 |
| **2030** | 神经形态芯片 | 类脑计算，超低功耗 |

### 6.2 国产化进程

| 阶段 | 时间 | 目标 | 进展 |
|-----|------|------|------|
| **替代** | 2020-2025 | 进口芯片替代 | 地平线J5、黑芝麻A2000量产 |
| **并行** | 2025-2028 | 国产与进口共存 | 华为MDC系列扩大应用 |
| **领先** | 2028+ | 部分领域领先 | 在算法-芯片协同优化上突破 |
